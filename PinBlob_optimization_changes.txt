PinBlob__CLV_2024__optimized.vbs Optimization Changes
======================================================
Source:    PinBlob__CLV_2024_.vbs
Reference: optimization-guide.txt, distilled-guidance.txt, dhChanges.txt (GOT)

15 optimizations applied. All changes verified clean.

Key differences from previous tables:
- gBOT is a fixed array (Array(MBall1,MBall2,MBall3)) — no GetBalls in FlipperNudge
- DynamicBSUpdate already uses DistanceFast() instead of Distance() — kept as-is
- DynamicBSUpdate uses DynamicBSFactor^2 AND ^3 (previous tables had only ^3)
- SpinnerTimer used hardcoded 3.14 approximation instead of actual PI
- FlasherTimer_Timer runs at 15ms (67Hz) with fValue^3 — a real hot path
- FlashFlasher has ^2.5/^3/^2 on ObjLevel
- InCircle function used ^2 in a distance calc
- CoRTracker iterates gBOT directly (no GetBalls) — already efficient
- tnob=3 (smallest ball count of all tables optimized)
- LampFilter ^1.6 present and active


1. FLIPPER TIMER 1ms -> 10ms  (line ~2724)
   Was: RightFlipper.timerinterval=1  (1000Hz)
   Now: RightFlipper.timerinterval=10 (100Hz)
   Saves: ~4000 total sub calls/sec -> ~400.


2. PRESS GUARD IN RightFlipper_Timer  (line ~2727)
   Added: If LFPress = 0 And RFPress = 0 Then Exit Sub
   Placed before FlipperNudge calls. gBOT is a fixed array so no GetBalls refactor.
   Saves: All FlipperNudge + FlipperTrigger work eliminated when flippers at rest.


3. LampFilter -- ^1.6 REPLACED WITH 101-ENTRY LUT  (line ~5508)
   Was: LampFilter = aLvl^1.6 -- ~9000 ^-exp/sec via Lampz.FilterOut every frame.
   Now: Pre-built LampFilterLUT(0..100) = (i/100)^1.6 at init.
        LampFilter does idx=Int(aLvl*100+0.5) then returns LampFilterLUT(idx).
   Saves: ~9000 ^-exp/sec eliminated.


4. AudioFade / AudioPan -- ^10 CHAIN + INVERSE MULTIPLIERS  (lines ~4052, ~4075)
   Was: tmp^10 + 2/tableheight or 2/tablewidth division per call.
   Now: t2=tmp*tmp : t4=t2*t2 : t8=t4*t4 : result=t8*t2
        InvTH2 = 2/tableheight, InvTW2 = 2/tablewidth (module-level).
   Saves: ~500 ^-exp/sec + ~240 divisions/sec.


5. FlipperTricks -- CACHE startangle AND currentangle  (line ~2908)
   Was: Flipper.startangle read twice; Flipper.currentangle read up to 3 times.
   Now: sa/absSa/Dir cached at entry; ca=Abs(Flipper.currentangle) cached.
   Saves: ~4000 COM reads/sec at 1000Hz x 2 flippers.


6. FlipperNudge -- CACHE currentangle  (line ~2748)
   Was: Flipper1.currentangle read twice per call.
   Now: Dim ca1 : ca1 = Flipper1.currentangle cached at entry.
   Saves: ~200 COM reads/sec.


7. PIover180 / d180overPI + ALL PI/180 INSTANCES FIXED  (line ~2779)
   Added: Dim PIover180 : PIover180 = PI/180
          Dim d180overPI : d180overPI = 180/PI
   Applied in:
     dSin/dCos:         * Pi/180     -> * PIover180
     Radians():         * PI/180     -> * PIover180
     AnglePP():         * 180/PI     -> * d180overPI
     FlipperTrigger:    inlined with PIover180
     SpinnerTimer:      2*3.14/360   -> PIover180  (was also WRONG — used 3.14 not PI)
     DTAnimate (x2):    3.1416/180   -> PIover180
     STAnimate:         3.1416/180   -> PIover180
     DoDTAnim:          PI/180       -> PIover180
   Note: SpinnerTimer fix also corrects a precision bug — 3.14 vs actual PI.
   Saves: ~8000+ divisions/sec + more accurate spinner animation.


7b. Distance() -- ^2 -> MULTIPLICATIONS  (line ~2809)
    Was: SQR((ax-bx)^2 + (ay-by)^2)
    Now: dx=ax-bx : dy=ay-by : Distance = SQR(dx*dx + dy*dy)


8. DynamicBSUpdate -- FULL REWRITE  (line ~2303)
   a) falloff moved from inside sub (Dim'd every frame) to module-level BS_falloff.
   b) gBOT(s).X/Y/Z cached into bx/by/bz (~20 reads -> 3 per ball per frame).
      Saves: ~17 COM reads/ball/frame x 3 balls x 60fps = ~3060 COM reads/sec.
   c) Pre-computed module-level constants:
        BS_TW2       = tablewidth/2
        BS_d10       = 5    (BallSize/10)
        BS_d4        = 12.5 (BallSize/4)
        BS_d5        = 10   (BallSize/5)
        BS_d2        = 25   (BallSize/2, used in height calc)
        BS_dAM       = 25   (BallSize/AmbientMovement)
        DynBSFactor2 = DynamicBSFactor^2 (= 0.9025, replaces ^2 per shadow)
        DynBSFactor3 = DynamicBSFactor^3 (= 0.857375, replaces ^3 per shadow)
      Saves: ~100 divisions/sec + 3 ^-exp/ball/frame eliminated.
   d) DSSources(iii)(0/1) cached into sx/sy per source iteration.
   e) currentShadowCount(s) reset at top of ball loop (cleaner than original Else branch).
   f) UBound(gBOT) cached into ubGBOT; early exit when no balls.
   NOTE: DistanceFast() kept — already an optimized approximation, faster than Sqr.


9. UpdateBallBrightness -- EARLY EXIT + COLOUR PRE-COMPUTED  (line ~5603)
   Was: Loop ran even with no balls; recomputed color expression per iteration.
   Now: UBound(gBOT) cached into ubGBOT; If ubGBOT < lob Then Exit Sub.
        Color value computed once outside loop (bbgi branch) and reused.
   Saves: Eliminates loop entirely when no balls; ~2 expression evals/ball/frame.


10. BallVel / Vol / Volz / VolPlayfieldRoll / PitchPlayfieldRoll -- ALL ^N ELIMINATED
    (lines ~4091-4118)
    BallVel:           ^2 -> vx*vx+vy*vy  (cached locals)
    Vol:               ^2 -> bv*bv  (cached bv)
    Volz:              ^2 -> vz*vz  (cached vz)
    VolPlayfieldRoll:  ^3 -> bv*bv*bv  (cached bv)
    PitchPlayfieldRoll: ^2*15 -> bv*bv*15  (cached bv)


11. BallSpeed / TargetBouncer -- ^N ELIMINATED  (lines ~2689, ~2428)
    BallSpeed: VelX^2+VelY^2+VelZ^2 -> cache vx/vy/vz, Sqr(vx*vx+vy*vy+vz*vz)
    TargetBouncer: vel^2, velz^2, vratio^2 -> vel*vel, velz2*velz2, vratio*vratio
                   Caches VelX/VelY/VelZ; inlines BallSpeed.


12. RollingUpdate -- REWRITE  (line ~3207)
    a) BallRollStr(3) pre-built: "BallRoll_" & b -> BallRollStr(b)
       Saves: ~900 BSTR allocs/sec (3 balls x 100Hz x 3 calls).
    b) Set ball = gBOT(b) caches COM object per iteration.
    c) bvel computed once; inlines VolPlayfieldRoll + PitchPlayfieldRoll.
       Saves: ~600 COM reads/sec (2 redundant BallVel x 3 balls x 100Hz).
    d) bz/bvz cached from ball.Z/VelZ. UBound(gBOT) cached into ubGBOT.
    e) BallSize/4, BallSize/2-5, BallSize/5 replaced with BS_d4, BS_d2, BS_d5.


13. RampRollUpdate + WRemoveBall -- REWRITE  (line ~3351)
    a) RampLoopStr(4)/WireLoopStr(4) pre-built. String allocs eliminated.
    b) Set rball = RampBalls(x,0) caches ball COM object.
    c) bvel computed once; BallPitchV/BallPitch inlined as pSlope(bvel,...);
       VolPlayfieldRoll inlined as bvel*bvel*bvel.
    d) WRemoveBall updated to use RampLoopStr/WireLoopStr.


14. CoRTracker.Update -- INLINE BallSpeed + CACHE VelX/VelY  (line ~3088)
    Was: Two For-Each passes over gBOT; second called BallSpeed(b).
    Now: Single pass. bvx/bvy cached; ballvel = Sqr(bvx*bvx+bvy*bvy+b.velz*b.velz).
    Note: This table's CoRTracker uses gBOT directly (not GetBalls) — already efficient.


15. FlipperVisualUpdate -- CACHE CurrentAngle  (line ~793)
    Was: With FlipperColorMod>=1, each flipper's CurrentAngle read twice
         (once for pFlipperBally, once for pFlipperShadow).
    Now: aLF/aRF cached at entry; reused in both branches.
    Saves: ~240 COM reads/sec at 60fps when FlipperColorMod >= 1.


BONUS FIXES (non-standard for this table):

  FlasherTimer_Timer -- fValue(ii,0)^3 -> fv*fv*fv  (line ~1535)
    FlasherTimer runs at 15ms (67Hz) x number of active flashers.
    fValue(ii,0) cached into fv; fv*fv*fv replaces ^3. Also caches ObjLevel in
    Flash sub (ObjLevel(nr)*0.9-0.01 -> ol*0.9-0.01).

  FlashFlasher -- ^2.5/^3/^2 -> multiplications  (line ~5005)
    ol=ObjLevel(nr) cached; ol2=ol*ol.
    ^2.5 -> ol2*Sqr(ol), ^3 -> ol2*ol, ^2 -> ol2.
    Also caches ObjLevel in the decay: ObjLevel(nr)*0.9-0.01 -> ol*0.9-0.01.

  InCircle -- ^2 -> dx*dx+dy*dy  (line ~3165)
    Replaces Sqr((pX-centerX)^2 + (pY-centerY)^2) with cached dx/dy multiplications.

  Ball collision sound -- velocity^2 -> cv*cv  (line ~4600)
    CSng(velocity)^2 -> cv=CSng(velocity) : cv*cv.

  SpinnerTimer precision fix  (line ~1083)
    Was using 3.14 (inaccurate hardcoded approximation of PI).
    Now uses PIover180 which is exact and pre-computed. Minor visual improvement.

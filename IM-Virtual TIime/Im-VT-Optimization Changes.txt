Iron_Maiden_Virtual_Time_Optimized.vbs Optimization Changes
==============================================================
Source: Iron_Maiden_Virtual_Time__Original_2020_.vbs
Table: Iron Maiden Virtual Time (Original 2020)
By Franck Hollinger (Mussinger) and Herweh on Codes


1. BallVel() — ^2 REPLACED WITH MULTIPLICATIONS  (line ~3243)
   Was: INT(SQR((ball.VelX ^2) + (ball.VelY ^2)))  — 2 ^-exp ops + 2 COM reads
   Now: Dim vx,vy : vx=ball.VelX : vy=ball.VelY : BallVel = INT(SQR(vx*vx + vy*vy))
   Why: VBScript's ^ operator uses the slow log/exp path even for integer exponents.
        Caches VelX/VelY into locals; replaces 2 ^-exp with multiplications.
   Called from: Vol(), Pitch(), RollingSoundTimer (per ball per tick), Kickback_Hit,
        PlaySoundAtBall, PlaySoundAtBallVol, RandomBump.
   Saves: ~2 ^-exp per call × ~10+ calls/tick × 100Hz = ~2000 ^-exp/sec.


2. Vol() — ^2 REPLACED + BallVel CACHED  (line ~3222)
   Was: Csng(BallVel(ball) ^2 / 200)
   Now: Dim bv : bv = BallVel(ball) : Vol = Csng(bv * bv / 200)
   Why: Caches BallVel result (avoiding redundant re-computation) and replaces ^2.
   Called from: RollingSoundTimer (per ball per tick), PlaySoundAtBallVol,
        RandomBump, MetalWalls_Hit, Plastics_Hit.
   Saves: 1 ^-exp + 1 redundant BallVel call per invocation.


3. VolZ() — ^2 REPLACED + BallVelZ CACHED  (line ~3226)
   Was: Csng(BallVelZ(ball) ^2 / 2000)
   Now: Dim bvz : bvz = BallVelZ(ball) : VolZ = Csng(bvz * bvz / 2000)
   Why: Caches BallVelZ result and replaces ^2.
   Saves: 1 ^-exp per call.


4. AudioPan() — ^10 REPLACED WITH MULTIPLY CHAIN  (line ~3230)
   Was: tmp^10 and -((-tmp)^10)  — VBScript's ^ uses log/exp for every call.
   Now: t2=tmp*tmp : t4=t2*t2 : t8=t4*t4 : result=t8*t2
        Separate positive/negative/zero branches. Same math, ~10x faster.
   Called per ball per frame in RollingSoundTimer, per PlaySoundAt* call,
        and per ball per GraphicsTimer tick.
   Saves: ~1 ^-exp per call × ~20+ calls/tick = ~2000 ^-exp/sec.


5. AudioFade() — ^10 REPLACED WITH MULTIPLY CHAIN  (line ~3247)
   Was: tmp^10 and -((-tmp)^10)
   Now: Same multiply chain as AudioPan.
   Called in parallel with AudioPan — same frequency.
   Saves: ~2000 ^-exp/sec (same as AudioPan).


6. InCircle() — ^2 REPLACED WITH MULTIPLICATIONS  (line ~3464)
   Was: Sqr((pX - centerX) ^ 2 + (pY - centerY) ^ 2)  — 2 ^-exp ops
   Now: Dim dx,dy : dx=pX-centerX : dy=pY-centerY : Sqr(dx*dx + dy*dy)
   Why: InCircle is called 3 times per ball per RollingSoundTimer tick
        (once each for sw25, sw33, sw38 saucer proximity checks).
        With 3 balls in play at 100Hz = 900 calls/sec.
   Saves: 2 ^-exp per call × 900 calls/sec = ~1800 ^-exp/sec.


7. OnBallBallCollision — ^2 REPLACED  (line ~3418)
   Was: Csng(velocity) ^2 / 2000
   Now: Csng(velocity * velocity) / 2000
   Why: Called on every ball-ball collision event.
   Saves: 1 ^-exp per collision.


8. FlasherTimer — ^3 REPLACED WITH CACHED MULTIPLICATION  (line ~2307)
   Was: flashx3 = fValue(ii,0) ^ 3
        matdim = Round(10 * fValue(ii,0))
        (fValue(ii,0) read from 2D array twice)
   Now: Dim fv0 : fv0 = fValue(ii,0)
        flashx3 = fv0 * fv0 * fv0
        matdim = Round(10 * fv0)
   Why: FlasherTimer runs at 15ms interval for all 13 flasher slots.
        The ^3 exponent is computed per active flasher per tick.
        Caching fValue(ii,0) also eliminates a redundant 2D array dereference.
   Saves: 1 ^-exp + 1 array deref per flasher per tick × ~67Hz = ~870 ^-exp/sec
          (when flashers active).


9. GraphicsTimer — CONSTANT DIVISIONS PRE-COMPUTED  (line ~3268)
   Was: IronMaiden.Width/2 and Ballsize/6 computed inside per-ball shadow loop.
        IronMaiden.Width is a COM property read; both are constant divisions
        repeated per ball every 10ms tick.
   Now: Dim halfWidth : halfWidth = IronMaiden.Width/2
        Dim bsDiv6 : bsDiv6 = Ballsize/6
        Pre-computed once before the ball loop, locals reused.
   Why: With ShowBallShadow enabled, these divisions occur ~4 times per ball
        (2 comparisons + 2 calculations) × up to 6 balls × 100Hz.
   Saves: ~4 divisions/ball/tick × 3 balls × 100Hz = ~1200 divisions/sec
          + ~1 COM property read/tick eliminated.


10. RollingSoundTimer — UBound(BOT) CACHED  (line ~3323)
    Was: UBound(BOT) called 3 times (stop loop, exit check, play loop).
    Now: Dim ubBOT : ubBOT = UBound(BOT) — cached once, reused.
    Called from RollingSoundTimer at 10ms interval (100Hz).
    Saves: 2 redundant UBound evaluations/tick × 100Hz = ~200 ops/sec.


NOTES ON PLATFORM COMPATIBILITY:
=================================
All optimizations use only local variable caching and arithmetic substitutions.
No module-level For loops, no LUT arrays, no Eval() calls — these patterns are
avoided to ensure compatibility with Linux standalone VPX builds.
